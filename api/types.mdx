### PermitToken

A cryptographically random token that enables trusted restarts without Telegram approval.

```typescript
type PermitToken = string | null
```

**Properties**:
- Format: 64-character hex string (32 bytes)
- Generated by: Worker on approval (SDK/src/telegram.js:170)
- Stored in: RAM only (module-scoped in vault.js:17)
- Expiration: 24 hours (TTL in KV storage)

**Source**: SDK/src/vault.js:17

### VaultData

Raw encrypted vault content in `.env` format.

```typescript
type VaultData = string
```

**Format**:
```bash
DATABASE_URL=ENC[AES256_GCM,data:...,iv:...,tag:...,type:str]
API_KEY=plaintext-value
SECRET_TOKEN=ENC[AES256_GCM,data:...,iv:...,tag:...,type:str]
```

**Encryption Format**:
- Prefix: `ENC[AES256_GCM,`
- Fields: `data:`, `iv:`, `tag:`, `type:`
- Encoding: Base64

**Source**: SDK/src/vault-file.js

### MasterKey

Reconstructed AES-256-GCM encryption key from two halves.

```typescript
interface MasterKey {
  /** Full 64-character hex string (32 bytes) */
  key: string
}
```

**Construction**:
```javascript
const masterKey = joinKey(convexHalf, cfHalf)
// convexHalf: 32 chars (16 bytes hex)
// cfHalf:     32 chars (16 bytes hex)  
// masterKey:  64 chars (32 bytes hex)
```

**Source**: SDK/src/crypto.js, SDK/src/vault.js:33

## Worker API Types

### AppStatus

Current approval and runtime status of an application.

```typescript
type AppStatus = 
  | 'idle'      // Initial state, no access requested
  | 'pending'   // Access requested, awaiting admin approval
  | 'approved'  // Approved but secrets not yet loaded
  | 'active'    // Secrets loaded and running
  | 'denied'    // Access denied or revoked
```

**Transitions**:
- `idle` → `pending`: Access requested via `/request`
- `pending` → `approved`: Admin approves in Telegram
- `pending` → `denied`: Admin denies in Telegram
- `approved` → `active`: Secrets loaded via `/loaded`
- `active` → `denied`: Admin revokes access

**Source**: worker/src/index.js, worker/src/telegram.js:41-46

### ActivityAction

Types of logged activities.

```typescript
type ActivityAction =
  | 'initialized'      // App created via suron init
  | 'access_requested' // Server requested access
  | 'approved'         // Admin approved access
  | 'denied'          // Admin denied access
  | 'revoked'         // Admin revoked active access
  | 'stopped'         // Admin stopped app
  | 'secrets_loaded'  // Secrets loaded into memory
  | 'key_rotated'     // Master key rotated
  | 'vault_uploaded'  // Vault data updated
  | 'resumed_with_permit' // Trusted restart succeeded
```

**Source**: worker/src/telegram.js:48-60

### KeyHalves

Split master key stored across two services.

```typescript
interface KeyHalves {
  /** First half stored in Convex (16 bytes hex) */
  convexHalf: string
  
  /** Second half stored in Cloudflare KV (16 bytes hex) */
  cfHalf: string
}
```

**Security**:
- Each half is 32 hex characters (16 bytes)
- Neither half can decrypt data alone
- Both required to reconstruct master key

**Source**: worker/src/index.js:132-145

## Encryption Types

### EncryptedValue

Format of an encrypted secret value.

```typescript
interface EncryptedValue {
  /** Encryption algorithm */
  algorithm: 'AES256_GCM'
  
  /** Base64-encoded encrypted data */
  data: string
  
  /** Base64-encoded initialization vector */
  iv: string
  
  /** Base64-encoded authentication tag */
  tag: string
  
  /** Value type (always 'str' for now) */
  type: 'str'
}
```

**Serialized Format**:
```
ENC[AES256_GCM,data:base64...,iv:base64...,tag:base64...,type:str]
```

**Source**: SDK/src/crypto.js, CLI/src/crypto.js
