## Prerequisites

All examples require these 3 environment variables on the server:

```env
VAULT_WORKER_URL=https://suronai-vault.your-account.workers.dev
VAULT_ACCESS_TOKEN=your_server_access_token
VAULT_APP_ID=my-app
```

Set them in `.env` locally, or as environment variables on your server/platform.

## Quick Start

```bash
# 1. Set up infrastructure (once)
suron login
suron init --app my-app
suron encrypt          # encrypts your .env → .env.suron and pushes

# 2. Install SDK in your project
npm install @suronai/sdk

# 3. Add vault.load() as the very first line in your app
# See any example below
```

## How Approval Works

When your app starts, you'll receive a Telegram message:

```
Suron — Vault Access Request

App:          my-app
Host:         your-server-hostname
Last active:  2h ago

[ ✅ Approve ]  [ ❌ Deny ]
```

Tap **Approve** and your app decrypts its secrets and continues starting up.

After the first approval, a **permit token** is issued. If you persist it, restarts skip Telegram automatically for 24 hours. See [Trusted Restart](/examples/trusted-restart).
