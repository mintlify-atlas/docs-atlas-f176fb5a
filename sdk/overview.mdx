## Introduction

The `@suronai/sdk` package provides a simple, secure way to load encrypted secrets into your Node.js application. All secrets are approved via Telegram and decrypted directly into `process.env`.

## Installation

```bash
npm install @suronai/sdk
```

## Basic Usage

The SDK exports a single `vault` object with all the methods you need:

```javascript
import { vault } from '@suronai/sdk'

// Load secrets — blocks until approved via Telegram
await vault.load()

// Secrets are now available in process.env
console.log('API key loaded:', process.env.API_KEY ? 'yes' : 'no')
console.log('Database URL:', process.env.DATABASE_URL)
```

## Required Environment Variables

Before calling `vault.load()`, you must set these environment variables in your server's `.env` file:

<ParamField path="VAULT_WORKER_URL" type="string" required>
  URL of your Suron Vault worker (e.g., `https://your-worker.workers.dev`)
</ParamField>

<ParamField path="VAULT_ACCESS_TOKEN" type="string" required>
  Access token for authenticating with the vault worker. Generated by the Suron CLI.
</ParamField>

<ParamField path="VAULT_APP_ID" type="string" required>
  Unique identifier for your application. Used to identify which vault to load.
</ParamField>

<ParamField path="VAULT_PERMIT" type="string">
  Optional. Permit token for trusted restarts. See [Trusted Restart](/sdk/trusted-restart) for details.
</ParamField>

## SDK Exports

The SDK exports a `vault` object with the following methods:

```javascript
import { vault } from '@suronai/sdk'

// Available methods:
vault.load()           // Load secrets with Telegram approval
vault.hotReload()      // Internal — called automatically by heartbeat
vault.getPermitToken() // Get current permit token for trusted restarts
vault.events           // EventEmitter for lifecycle events
```

## Approval Flow

When you call `vault.load()` for the first time:

1. SDK sends a request to the vault worker
2. Admin receives a Telegram notification with server hostname
3. SDK polls for approval (5 minute timeout)
4. Admin taps **Approve** in Telegram
5. SDK decrypts secrets into `process.env`
6. Heartbeat starts to detect secret updates

<Tip>
  After initial approval, use [Trusted Restart](/sdk/trusted-restart) to skip Telegram approval for 24 hours.
</Tip>
